# docs/ — everything that supports the canon but isn't the canon

The canon itself (`specline-2.7.md`) and its readable companion (`handbook.md`)
live at the repo root. This tree holds the supporting material, so root stays
clean.

| Folder | What's here |
|---|---|
| `research/` | Evidence and positioning — `findings.md` (decisions from real runs), the thesis, and prior-art comparisons. The thinking behind the canon. |
| `proposals/` | Amendment shaping records — one per canon cluster (`v2.4`, `v2.5`). Superseded once folded into the canon; kept for provenance. |
| `history/` | Prior canon lineage (`spec-flow` v1, v2), kept for the audit trail. |
| `conventions/` | Templates and standards a Specline-governed repo adopts (e.g. the `CLAUDE.md` template). |

Each folder carries its own `README.md` — the canon's self-documenting-folders
convention, dogfooded here.
